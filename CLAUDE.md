# WATT Automation

Single-client content automation platform for a specific publication. Uses n8n for content generation workflows.

## Architecture

```
Publication (single tenant)
  ├── News Sources (RSS, web, keyword search, etc.)
  ├── Candidate Articles (discovered by research pipeline)
  ├── News Content → Content Versions (generated by n8n workflows)
  ├── Newsletter Templates → Newsletters
  └── Scheduling config
```

**Stack:** Flask, PostgreSQL, Celery + Redis, n8n (external workflow engine), SQLAlchemy/Alembic.

**Deployment:** Heroku (Procfile: web, worker, clock, release). DB is on AWS RDS.

## How n8n Fits In

Flask handles research, scheduling, data management, and UI. Content generation is delegated to n8n workflows via fire-and-forget HTTP triggers. n8n calls back into the Flask API when done.

**5 n8n Workflows:**
1. **Content Generation** — fetches candidates, generates articles via LLMs, POSTs results to `/api/news`
2. **Candidate Content Generation** — converts selected candidates into articles
3. **Image Generation** — generates featured images, calls back to `/api/workflow/<id>/image-complete`
4. **Audit** — audits article versions, patches issues, calls back to `/api/version-audit` and `/api/patched-version`
5. **Submit URL** — processes a single URL through the content pipeline

Configured via env vars: `N8N_CONTENT_WORKFLOW_URL`, `N8N_IMAGE_WORKFLOW_URL`, etc.

| `auth` | `/` | Login, logout, register |

## Key Models (`app/models.py`)

- **Publication** — the single client's publication. Has scheduling, CMS/Ghost config, Sponsy ad config, notification emails.
- **NewsSource** — RSS feeds, sites, keyword configs.
- **CandidateArticle** — discovered articles with relevance/keyword/recency scoring. Status: new → selected → processed.
- **NewsContent** — article container with legacy fields + multi-version support.
- **ContentVersion** — one per LLM (ai_provider, ai_model, quality_score, content fields, CMS/Ghost push tracking).
- **VersionAudit** — audit findings (issues JSON, overall_risk).
- **PatchedVersion** — editor bot patched draft.
- **WorkflowRun** — tracks n8n workflow execution (UUID passed to n8n, callbacks update status).
- **NewsletterTemplate / Newsletter / NewsletterItem** — newsletter assembly.

## Research Pipeline (`app/research/`)

Runs natively in Flask (not n8n):
- Scrapers: RSS, news sites, keyword search (SerpAPI), YouTube, competitors
- Dedup by URL hash
- Optional LLM triage via Claude Haiku
- Score by keyword relevance + recency + source weight
- Enrich high-scoring candidates via Firecrawl

Triggered hourly by Celery Beat or manually.

## Integrations

- **Ghost CMS** (`app/ghost.py`) — push articles/newsletters
- **Sponsy** (`app/sponsy.py`) — ad block/placement fetching for newsletters (uses `GET /ad-blocks` and `POST /ad-blocks/html` endpoints, NOT `/v1/` prefix)
- **Mailgun** (`app/notifications.py`) — job completion emails for research, content gen, candidate content gen
- **Firecrawl** — candidate enrichment
- **SerpAPI** — keyword search source type

## Background Tasks (`app/tasks.py`, `app/celery.py`)

Celery Beat schedule:
- `check_publication_schedules` — every 60s, triggers n8n content generation
- `check_candidate_content_schedules` — every 60s, triggers n8n candidate content gen
- `check_research_schedules` — every hour, runs native research pipeline

## API Authentication

Two auth levels via `X-API-Key` header:
- **Global:** `N8N_API_KEY` config → full access to all endpoints
- **Per-publication:** `publication.access_api_key` → restricted to that publication's data

## Config (`config.py`)

Key env vars:
- `DATABASE_URL` — PostgreSQL (RDS)
- `N8N_API_KEY` — global API auth + n8n webhook auth
- `N8N_*_WORKFLOW_URL` — 5 workflow trigger URLs
- `ANTHROPIC_API_KEY` — for LLM triage in research
- `FIRECRAWL_API_KEY`, `SERPAPI_API_KEY` — research pipeline
- `REDIS_URL` — Celery broker
- `MAILGUN_API_KEY`, `MAILGUN_DOMAIN` — notifications
